# 伏笔管理器

管理小说创作中的伏笔生命周期：创建 → 推进 → 回收。

## 伏笔生命周期

```
[创建] ──→ [pending] ──→ [advanced] ──→ [resolved]
              │              │              │
              │              │              └── 已回收
              │              └── 已推进
              └── 待处理（埋设后等待）
```

## 操作类型

### 创建伏笔

**流程**：询问基本信息 → 生成ID（fs_XXX） → 使用模板创建文件

**必填**：伏笔名称、埋设章节、相关角色ID

**可选**：重要性（high/medium/low）、预期回收章节、回收方式

### 查询伏笔

**维度**：按状态/章节/角色/重要性

### 更新伏笔状态

**状态转换**：
- `pending → advanced`（推进）
- `advanced → resolved`（回收）
- `pending → resolved`（直接回收）

### 删除伏笔

删除前需确认，已回收的伏笔不建议删除。

## 文件结构

```
状态追踪/伏笔管理/
├── fs_001_林家灭门真相.md
├── fs_002_神秘功法来源.md
└── ...
```

## 模板

使用 `templates/伏笔模板.md` 创建新伏笔文件。

模板包含：frontmatter元数据、埋设方式、预期回收、推进记录、回收记录。

## 上下文集成

### 章节生成时

```markdown
【待处理伏笔】
- fs_001 林家灭门真相（pending，第1章埋设）
  摘要：林叶家族灭门的真相，涉及宗门高层
```

### 章节分析时

自动识别：新埋设 → 创建文件 | 推进 → 更新记录 | 回收 → 更新状态

## 协作

| Skill | 协作方式 |
|-------|----------|
| context-manager | 构建上下文时读取伏笔信息 |
| outline-generator | 规划大纲时关联伏笔 |
