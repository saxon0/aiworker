# 小说章节生成器

基于RTCO框架的小说章节生成工具，支持上下文构建、智能分场规划和内容去重验证。

## 快速开始

```bash
python scripts/generate_chapter.py --chapter <章节号> --project <项目目录>
```

## 核心功能

### 1. 生成前预检

执行章节生成前自动检查：
- 大纲是否存在及质量评分
- 前文衔接完整性
- 角色信息完整性
- 世界观设定完整度

### 2. RTCO上下文构建（P0+P1）

**P0 核心信息（必须包含）**：
- 章节大纲
- 衔接锚点（上一章结尾）

**P1 重要信息（按预算动态调整）**：
- 角色信息
- 角色关系网络
- 角色属性/状态
- 组织势力
- 世界观设定
- 角色动态状态（第3章起）
- 故事时间线（第5章起）
- 因果链上下文（第5章起）

### 3. 智能分场规划

将章节拆分为3-5个场景，每个场景包含：
- 地点 (location)
- 出场人物 (characters)
- 主要动作 (action)
- 核心冲突 (conflict)
- 结果/钩子 (outcome)

### 4. 内容去重验证

生成完成后执行：
- 前3章文本相似度检测
- N-gram重叠分析
- 自动修复建议

## 使用方法

### 基本用法

```bash
# 生成第10章
python scripts/generate_chapter.py --chapter 10 --project ./my-novel

# 指定目标字数
python scripts/generate_chapter.py --chapter 10 --project ./my-novel --words 3000

# 指定节奏等级(1-5)
python scripts/generate_chapter.py --chapter 10 --project ./my-novel --pace 4

# 指定本章目标
python scripts/generate_chapter.py --chapter 10 --project ./my-novel --goal "主角在拍卖会上遇到宿敌"
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--chapter` | 章节号（必填） | - |
| `--project` | 项目目录（必填） | - |
| `--words` | 目标字数 | 2500 |
| `--pace` | 节奏等级(1-5) | 3 |
| `--goal` | 本章目标 | "延续剧情发展" |
| `--output` | 输出目录 | 项目目录/正文 |

## 注意事项

1. **大纲优先**：没有大纲时会提示用户先创建大纲
2. **衔接锚点**：第2章起必须加载上一章结尾作为衔接
3. **字数控制**：目标字数的85%-120%为合理区间
4. **去重验证**：生成后自动检测，不阻塞主流程
