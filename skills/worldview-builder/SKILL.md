# 世界观构建器

引导用户完成小说世界观的构建：专家讨论配置 → 检索现有项目 → 确认目标项目 → 填充详细信息 → 生成世界观文件。

## 工作流程

```
用户请求创建世界观
       ↓
┌──────────────────┐
│ 0. 专家讨论配置   │  ← 调用 project-discussion skill 讨论世界观配置
└──────────────────┘
       ↓
┌──────────────────┐
│ 1. 检索现有项目   │  ← 扫描工作目录下的小说项目
└──────────────────┘
       ↓
┌──────────────────┐
│ 2. 确认目标项目   │  ← 询问用户选择或指定项目
└──────────────────┘
       ↓
┌──────────────────┐
│ 3. 确认世界观类型  │  ← 根据小说类型确定世界观框架
└──────────────────┘
       ↓
┌──────────────────┐
│ 4. 填充世界观要素  │  ← 逐项填写世界观模板
└──────────────────┘
       ↓
┌──────────────────┐
│ 5. 生成世界观文件  │  ← 创建完整世界观文档（含YAML元数据）
└──────────────────┘
       ↓
   世界观就绪
```

## 第零步：专家讨论配置

**必须首先执行此步骤！**

调用 `project-discussion` skill，聘请世界级专家对 `templates/世界观模板.md` 中的所有信息进行深度讨论。

### 讨论议题

讨论内容应覆盖世界观模板中的所有要素：

| 讨论维度 | 配置项 | 讨论要点 |
|----------|--------|----------|
| 时空设定 | 时代、地点 | 时间线构建、历史背景深度、空间布局合理性 |
| 世界基调 | 氛围基调 | 整体氛围营造、情感基调、读者沉浸感 |
| 规则体系 | 规则 | 核心规则设计、规则自洽性、规则与剧情的互动 |
| 社会架构 | 社会结构 | 势力博弈、资源分配、信仰体系 |
| 能力体系 | 科技/超自然体系 | 体系创新性、能力来源逻辑、约束与代价平衡 |
| 文化深度 | 文化风俗 | 习俗设计、禁忌设定、文化冲突 |

### 执行方式

```
调用 project-discussion skill
     ↓
展示拟聘请专家名单
     ↓
用户确认专家
     ↓
执行多轮讨论
     ↓
输出讨论纪要
     ↓
基于讨论结果进入第一步
```

### 专家建议方向

根据世界观构建领域，建议聘请：
- **世界构建权威**：如 J.R.R. Tolkien（中土世界创造者）、Brandon Sanderson（魔法体系设计大师）
- **类型专家**：根据小说类型聘请对应领域专家（玄幻/科幻/历史等）
- **社会学专家**：社会结构、势力关系设计
- **资深读者代表**：目标读者群体的典型代表
- **顶级纪要员**：专业记录整理

**注意**：必须等待用户确认专家名单后才能开始讨论！

## 第一步：检索现有项目

### 项目识别标准

扫描工作目录，识别符合以下条件的小说项目：

1. **目录结构特征**：
   - 包含 `项目信息/项目配置.md` 文件
   - 或包含 `设定/世界观/` 目录
   - 或包含 `正文/` 目录

2. **识别方法**：
   - 使用 Glob 工具搜索 `**/项目配置.md`
   - 使用 LS 工具列出工作目录下的文件夹
   - 检查每个文件夹是否包含小说项目特征

### 项目信息提取

对于每个识别到的项目，提取以下信息：
- 项目名称（目录名）
- 小说类型（从项目配置中读取）
- 当前世界观状态（是否已有世界观文件）

## 第二步：确认目标项目

### 询问策略

使用 AskUserQuestion 工具询问用户：

1. **如果找到现有项目**：
   - 列出所有找到的项目供用户选择
   - 提供"新建项目"选项
   - 提供"指定其他路径"选项

2. **如果未找到项目**：
   - 询问用户是否要新建项目
   - 或指定已有项目的路径

### 选项格式示例

```
问题：请选择要创建世界观的项目

选项：
- 苍穹之巅（玄幻）- 已有世界观
- 星辰大海（科幻）- 无世界观
- 新建项目
- 指定其他项目路径
```

### 用户指定项目

用户可以通过以下方式指定项目：
- 直接说出项目名称
- 提供项目路径
- 描述项目特征（如"那个玄幻小说"）

## 第三步：确认世界观类型

根据项目配置中的小说类型确定世界观框架重点：

| 小说类型 | 世界观重点 |
|----------|------------|
| 玄幻/仙侠 | 修炼体系、境界划分、势力分布、天地法则 |
| 都市 | 社会背景、势力关系、特殊设定（如有） |
| 科幻 | 科技水平、星际格局、文明形态 |
| 历史 | 时代背景、政治格局、社会风貌 |
| 悬疑 | 社会环境、人物关系网、隐藏秘密 |
| 奇幻 | 魔法体系、种族设定、世界结构 |

## 第四步：填充世界观要素

读取 `templates/世界观模板.md`，逐项引导用户填写：

### 核心要素（必填）

| 要素 | 说明 | 示例 |
|------|------|------|
| 时代 | 时间线与历史背景 | 上古纪元、现代都市、星际时代 |
| 地点 | 主要区域与关键地点 | 九州大陆、东海城、青云宗 |
| 规则 | 世界运行的核心规则 | 修炼法则、魔法定律、社会法律 |

### 扩展要素（建议填写）

| 要素 | 说明 |
|------|------|
| 氛围基调 | 整体氛围与基调 |
| 社会结构 | 政治/势力、经济/资源、宗教/信仰 |
| 科技/超自然体系 | 体系名称、能力来源、约束与代价 |
| 文化风俗 | 语言/习俗、禁忌 |

**执行方式**：使用 AskUserQuestion 工具逐项收集信息，或让用户直接提供完整设定。

## 第五步：生成世界观文件

### 目录创建

**在写入文件前，必须确保目录存在：**

```
目标目录：{项目目录}/设定/世界观/

创建逻辑：
1. 检查 {项目目录}/设定/ 是否存在
2. 如不存在，创建 设定/ 目录
3. 检查 {项目目录}/设定/世界观/ 是否存在
4. 如不存在，创建 世界观/ 目录
5. 写入文件
```

### 文件创建规则

1. **存储位置**：
   - 存储到 `{项目目录}/设定/世界观/`

2. **文件命名**：
   - 主世界观：`世界观-主世界.md`
   - 特定区域：`世界观-{区域名}.md`
   - 特定体系：`世界观-{体系名}.md`

3. **文件内容**：
   - 基于模板填充用户设定
   - 包含 YAML 元数据（见下方规范）
   - 保持结构清晰、层次分明

## YAML 元数据规范

### 元数据结构

```yaml
---
# === 核心字段（总是读取）===
type: world-setting
category: {时代背景/地理区域/规则体系/势力组织/其他}
name: {名称}
scope: {global/volume/chapter}

# === 索引字段（过滤时读取）===
volumes: [{涉及的卷}]
locations: [{包含的地点}]

# === 摘要字段（快速了解）===
summary: {一句话摘要}
---
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 `world-setting` |
| category | string | 是 | 分类：时代背景/地理区域/规则体系/势力组织/其他 |
| name | string | 是 | 世界观名称 |
| scope | string | 是 | 作用域：global（全局）/volume（卷级）/chapter（章节级） |
| volumes | array | 否 | 涉及的卷号列表 |
| locations | array | 否 | 包含的地点列表 |
| summary | string | 是 | 一句话摘要，用于其他 Skill 快速了解 |

### 元数据用途

| 字段 | 被谁读取 | 读取目的 |
|------|----------|----------|
| type | 所有 Skill | 确认文件类型 |
| category | 大纲生成器 | 筛选特定类型的世界观 |
| name | 所有 Skill | 标识世界观 |
| scope | 大纲生成器 | 判断是否需要读取 |
| volumes | 大纲生成器 | 筛选当前卷相关的世界观 |
| locations | 场景生成器 | 匹配场景地点 |
| summary | 所有 Skill | 快速了解内容，决定是否深度读取 |

## 渐进式披露读取策略

其他 Skill 读取世界观文件时，应遵循以下策略：

### 阶段1：扫描（~50字符）

```
读取：type, name, category
目的：确认是世界观文件，了解类型
决策：是否需要继续读取
```

### 阶段2：过滤（~100字符）

```
读取：scope, volumes
目的：判断是否与当前任务相关
决策：是否需要此世界观
```

### 阶段3：关联（~150字符）

```
读取：locations
目的：建立地点关联
决策：需要关联哪些其他文件
```

### 阶段4：深度（全文）

```
读取：Markdown 正文
目的：获取完整世界观设定
决策：如何使用这些信息
```

## 完成确认

世界观创建完成后，向用户展示：

1. **项目信息**：关联的项目名称和类型
2. **世界观概览**：核心设定摘要
3. **文件位置**：生成的文件路径
4. **下一步建议**：
   - 创建人物档案
   - 完善势力设定
   - 规划故事大纲

## 注意事项

- 世界观设定要自洽，避免矛盾
- 保持适当留白，为后续创作预留空间
- 重要设定要有伏笔和展开空间
- 如果项目已有世界观，询问用户是覆盖还是补充
- **必须确保目录存在后再写入文件**
