# 小说项目创建器

引导用户完成小说项目的创建流程：专家讨论配置 → 确认项目配置 → 规划目录结构 → 创建项目文件。

## 工作流程

```
用户请求创建项目
       ↓
┌──────────────────┐
│ 0. 专家讨论配置   │  ← 调用 project-discussion skill 讨论项目配置
└──────────────────┘
       ↓
┌──────────────────┐
│ 1. 确认项目配置   │  ← 读取并填写项目配置模板
└──────────────────┘
       ↓
┌──────────────────┐
│ 2. 规划目录结构   │  ← 根据类型生成目录规划
└──────────────────┘
       ↓
┌──────────────────┐
│ 3. 创建项目文件   │  ← 初始化文件
└──────────────────┘
       ↓
    项目就绪
```

## 第零步：专家讨论配置

**必须首先执行此步骤！**

调用 `project-discussion` skill，聘请世界级专家对 `templates/项目配置模板.md` 中的所有信息进行深度讨论。

### 讨论议题

讨论内容应覆盖项目配置模板中的所有要素：

| 讨论维度 | 配置项 | 讨论要点 |
|----------|--------|----------|
| 作品定位 | 书名、主题、小说类型 | 市场定位、受众分析、类型选择策略 |
| 创作参数 | 每章字数、叙述视角、目标读者 | 写作节奏、叙事策略、读者画像 |
| 剧情设定 | 核心冲突、主要情节 | 故事张力、情节架构、冲突设计 |
| 风格选择 | 写作风格 | 风格匹配度、语言特色、表达方式 |

### 执行方式

```
调用 project-discussion skill
     ↓
根据项目需求动态选择专家
     ↓
展示拟聘请专家名单
     ↓
用户确认专家
     ↓
执行多轮讨论
     ↓
输出讨论纪要
     ↓
基于讨论结果进入第一步
```

### 专家选择原则

**根据具体项目动态决定专家人选，不预设固定专家角色！**

专家选择应考虑：
- **小说类型**：玄幻/科幻/都市/言情/历史等不同类型需要不同领域专家
- **目标读者**：不同读者群体的偏好
- **创作主题**：不同主题需要不同专业视角
- **用户需求**：根据用户的具体创作目标调整

**注意**：必须等待用户确认专家名单后才能开始讨论！

## 第一步：确认项目配置

读取 `templates/项目配置模板.md`，引导用户填写必填项：

### 必填项清单
| 字段 | 说明 | 示例 |
|------|------|------|
| 书名 | 作品标题 | 《星辰大海》 |
| 主题 | 核心主题 | 成长、复仇、爱情 |
| 小说类型 | 作品类型 | 玄幻/都市/言情/科幻/武侠/仙侠/历史/军事/悬疑/恐怖 |
| 每章目标字数 | 单章字数目标 | 2500（默认） |
| 叙述视角 | 叙事方式 | 第一人称/第三人称/混合视角 |

### 可选项
- 目标读者：青少年/成人/全年龄
- 核心冲突：故事核心矛盾
- 主要情节：故事主线走向

**执行方式**：使用 AskUserQuestion 工具逐项收集信息，或让用户直接提供完整配置。

## 第二步：规划目录结构

根据小说类型生成标准目录结构：

### 基础目录结构（v2.0 - 支持上下文管理）

```
{书名}/
│
├── 项目信息/                           # 【只读层】项目配置
│   ├── 项目配置.md                     # 书名、类型、风格等
│   ├── 创作宪法.md                     # 创作准则与规范
│   ├── 宪法维度.md                     # 创作维度与原则
│   └── 个人语料.md                     # 个人表达风格
│
├── 设定/                               # 【只读层】静态设定
│   ├── 世界观/
│   │   └── *.md
│   ├── 人物档案/
│   │   ├── 主角/
│   │   └── 配角/
│   ├── 组织势力/
│   └── 关系网络.md
│
├── 状态追踪/                           # 【读写层】动态数据
│   ├── 当前状态.md                     # 角色状态快照
│   ├── 伏笔管理/                       # 伏笔生命周期
│   └── 时间线.md                       # 故事时间线
│
├── 大纲/                               # 【写入层】故事规划
│   ├── 故事骨架.md                     # 整体故事线
│   └── 分卷大纲/
│       └── 第一卷/
│
├── 场景设计/                           # 【写入层】场景规划
│   └── 第一卷/
│
└── 正文/                               # 【写入层】章节产出
    └── 第一卷/
```

## 第三步：创建项目文件

### 文件创建顺序

1. **创建项目根目录**：`{书名}/`

2. **创建配置文件**：
   - 复制 `[skill目录]/templates/项目配置模板.md` → `项目配置.md`（填充用户配置）
   - 复制 `[skill目录]/templates/constitution.md` → `创作宪法.md`
   - 复制 `[skill目录]/templates/宪法维度.md` → `宪法维度.md`
   - 复制 `[skill目录]/templates/personal-voice.md` → `个人语料.md`

3. **创建设定目录**：
   - 创建 `设定/世界观/` 目录
   - 创建 `设定/人物档案/主角/` 目录
   - 创建 `设定/人物档案/配角/` 目录
   - 创建 `设定/组织势力/` 目录
   - 创建 `设定/关系网络.md` 文件（空模板）

4. **创建状态追踪目录**：
   - 创建 `状态追踪/` 目录
   - 创建 `状态追踪/当前状态.md` 文件（初始模板）
   - 创建 `状态追踪/伏笔管理/` 目录
   - 创建 `状态追踪/时间线.md` 文件（初始模板）

5. **创建大纲目录**：
   - 创建 `大纲/` 目录
   - 创建 `大纲/故事骨架.md` 文件（空模板）
   - 创建 `大纲/分卷大纲/第一卷/` 目录

6. **创建场景设计目录**：
   - 创建 `场景设计/第一卷/` 目录

7. **创建正文目录**：
   - 创建 `正文/第一卷/` 目录

8. **创建写作风格目录**：
   - 创建 `写作风格/` 目录
   - 复制 选定的和项目匹配的写作风格模板 → `写作风格/`

### 模板路径
```
[skill目录]/templates/
├── 项目配置模板.md
├── constitution.md
├── personal-voice.md
├── 当前状态模板.md           # 新增
├── 时间线模板.md             # 新增
└── 写作风格模板/        
```

### 状态追踪初始模板

**当前状态.md 初始内容**：
```markdown
---
type: state-snapshot
snapshot_chapter: 0
updated_at: {创建时间}
---

# 角色状态快照

<!-- 章节分析后自动更新此文件 -->
<!-- 格式：
## char_XXX
name: 角色名
current_state: "当前状态描述"
location: "当前位置"
emotional_state: "情绪状态"
recent_events:
  - chapter: N
    event: "事件描述"
-->
```

**时间线.md 初始内容**：
```markdown
---
type: timeline
created_at: {创建时间}
---

# 故事时间线

<!-- 章节分析后自动更新此文件 -->
<!-- 格式：
## 第N章
- 故事时间：{时间点}
- 关键事件：{事件描述}
-->
```

## 完成确认

项目创建完成后，向用户展示：

1. **项目结构概览**：目录树展示
2. **下一步建议**：
   - 完善世界观设定（调用 worldview-builder）
   - 创建主要人物档案（调用 character-builder）
   - 规划故事大纲（调用 outline-generator）

## 注意事项

- 目录不存在时候创建目录，如目录已存在，跳过创建步骤
- 保持模板文件的原始格式和注释说明
- 所有Markdown文件使用UTF-8编码
- 文件命名遵循ID前缀规范（见 references/directory-structure.md）
