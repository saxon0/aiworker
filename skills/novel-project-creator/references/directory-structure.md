# 小说项目目录结构参考

## 标准结构（v2.0 - 支持上下文管理）

```
{书名}/
│
├── 项目信息/                           # 【只读层】项目配置
│   ├── 项目配置.md                     # 书名、类型、风格等
│   ├── 创作宪法.md                     # 创作准则与规范
│   ├── 宪法维度.md                     # 创作维度与原则
│   └── 个人语料.md                     # 个人表达风格
│
├── 设定/                               # 【只读层】静态设定
│   ├── 世界观/
│   │   ├── 世界总览.md                 # 世界观总览
│   │   └── *.md                       # 各类世界观设定
│   │
│   ├── 人物档案/                       # 角色静态信息
│   │   ├── 主角/
│   │   │   └── char_XXX_角色名.md      # ID前缀命名
│   │   └── 配角/
│   │       └── char_XXX_角色名.md
│   │
│   ├── 组织势力/
│   │   └── org_XXX_组织名.md           # ID前缀命名
│   │
│   └── 关系网络.md                     # 角色关系网络
│
├── 状态追踪/                           # 【读写层】动态数据
│   ├── 当前状态.md                     # 所有角色当前状态快照
│   │
│   ├── 伏笔管理/                       # 伏笔生命周期
│   │   ├── fs_XXX_伏笔名.md            # ID前缀命名
│   │   └── ...
│   │
│   └── 时间线.md                       # 故事时间线
│
├── 大纲/                               # 【写入层】故事规划
│   ├── 故事骨架.md                     # 整体故事线
│   │
│   └── 分卷大纲/
│       └── 第N卷/
│           ├── ch_XXX_章节名.md        # ID前缀命名
│           └── ...
│
├── 场景设计/                           # 【写入层】场景规划
│   └── 第N卷/
│       ├── sc_XXX_YY_场景名.md         # ID前缀命名
│       └── ...
│
└── 正文/                               # 【写入层】章节产出
    └── 第N卷/
        ├── 第N章_章节名.md
        └── ...
```

## 目录层级说明

### 只读层（章节生成时只读取）

| 目录 | 用途 | 变化频率 | 上下文作用 |
|------|------|----------|------------|
| 项目信息/ | 项目配置、创作准则 | 极低 | P0核心信息 |
| 设定/世界观/ | 世界规则、设定 | 低 | P1重要信息 |
| 设定/人物档案/ | 角色静态信息 | 低 | P1重要信息 |
| 设定/组织势力/ | 组织静态信息 | 低 | P1重要信息 |
| 设定/关系网络/ | 角色关系 | 中 | P1重要信息 |

### 读写层（章节生成时读取，章节分析时更新）

| 目录 | 用途 | 变化频率 | 上下文作用 |
|------|------|----------|------------|
| 状态追踪/当前状态.md | 角色当前状态快照 | 高 | P1重要信息 |
| 状态追踪/伏笔管理/ | 伏笔生命周期 | 中 | P2参考信息 |
| 状态追踪/时间线.md | 故事时间线 | 中 | P2参考信息 |

### 写入层（创作产出）

| 目录 | 用途 | 变化频率 | 上下文作用 |
|------|------|----------|------------|
| 大纲/故事骨架.md | 整体故事线 | 低 | P2参考信息 |
| 大纲/分卷大纲/ | 章节大纲 | 中 | P0核心信息 |
| 场景设计/ | 场景规划 | 中 | P1重要信息 |
| 正文/ | 章节内容 | 高 | 衔接锚点来源 |

## 文件命名规范

### ID前缀命名规则

| 类型 | ID格式 | 命名规则 | 示例 |
|------|--------|----------|------|
| 角色 | char_XXX | char_{序号}_{角色名}.md | char_001_林叶.md |
| 组织 | org_XXX | org_{序号}_{组织名}.md | org_001_天澜宗.md |
| 章节 | ch_XXX | ch_{序号}_{章节名}.md | ch_001_入门.md |
| 场景 | sc_XXX_YY | sc_{章序号}_{场序号}_{场景名}.md | sc_001_01_入门试炼.md |
| 伏笔 | fs_XXX | fs_{序号}_{伏笔名}.md | fs_001_林家灭门真相.md |

### 章节文件命名

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 大纲文件 | ch_XXX_标题.md | ch_001_觉醒.md |
| 正文文件 | 第N章_标题.md | 第1章_觉醒.md |

## 渐进式披露支持

### frontmatter 必需字段

所有Markdown文件应包含以下frontmatter字段以支持渐进式披露：

```yaml
---
id: {类型前缀}_{序号}        # 唯一标识符
type: {文件类型}             # character/organization/chapter/scene/foreshadow
name: {名称}                 # 显示名称
summary: "{一句话摘要}"       # Level 0 摘要信息
tags: [标签1, 标签2]         # 用于相关性匹配
---
```

### 正文分层结构

```markdown
# 核心信息
<!-- Level 1 - 默认加载 -->

---
```

## 状态追踪目录详解

### 当前状态.md 结构

```yaml
---
type: state-snapshot
snapshot_chapter: {当前章节号}
updated_at: {更新时间}
---

name: 林叶
current_state: "筑基初期，伤势恢复中"

  - chapter: 9
    event: "发现宗门内奸线索"

## char_002
name: 楚天
current_state: "正常"
location: "天澜宗内门"
...
```

### 伏笔管理文件结构

```yaml
---
id: fs_001
type: foreshadow
name: 林家灭门真相
status: pending              # pending/advanced/resolved
importance: high             # high/medium/low
planted_chapter: 1           # 埋设章节
characters: [char_001]       # 相关角色ID
summary: "林叶家族灭门的真相"
---

# 伏笔内容

## 埋设方式
- 第1章：林叶回忆家族灭门，发现可疑之处

## 预期回收
- 计划章节: 第30-35章

## 推进记录
| 章节 | 推进内容 |
|------|----------|
| 10 | 发现新线索 |
```
